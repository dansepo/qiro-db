## 2. 스타일 가이드 개요
### 2.1. 목적
본 UX/UI 스타일 가이드는 QIRO 서비스의 디자인 원칙, 시각적 요소, 인터랙션 패턴, 공통 컴포넌트 등을 정의하여, 서비스 전반에 걸쳐 일관되고 높은 품질의 사용자 경험을 제공하는 것을 목적으로 한다. 또한, 기획, 디자인, 개발 과정에서의 효율적인 협업을 지원하고, 브랜드 아이덴티티를 강화한다.

### 2.2. 적용 대상
QIRO 서비스의 기획, 디자인, 개발에 참여하는 모든 팀원 및 외부 협력 파트너.

### 2.3. 핵심 디자인 원칙
- **명확성 (Clarity):** 사용자가 시스템의 기능과 정보를 쉽고 명확하게 이해할 수 있도록 한다.
- **일관성 (Consistency):** 서비스 전체에 걸쳐 통일된 디자인 언어와 인터랙션 패턴을 적용한다.
- **효율성 (Efficiency):** 사용자가 최소한의 노력으로 원하는 작업을 빠르고 효과적으로 완료할 수 있도록 지원한다.
- **사용자 중심 (User-Centered):** 실제 사용자의 요구와 작업 흐름을 최우선으로 고려하여 설계한다.
- **전문성 및 신뢰성 (Professional & Trustworthy):** 건물 관리 업무의 특성을 반영하여 전문적이고 신뢰감을 주는 디자인을 제공한다.
- **접근성 (Accessibility):** 다양한 사용자가 불편 없이 서비스를 이용할 수 있도록 웹 접근성 표준을 준수한다.

    - 각 페르소나의 주요 목표, 니즈, 사용 환경 특성 요약.

### 3.2. 정보 구조 (Information Architecture - IA)
- (요약 또는 IA 상세 문서 링크)
    - 주요 메뉴 구조 (GNB, LNB/Sidemenu).
    - 핵심 콘텐츠 및 기능의 네비게이션 흐름.
    - 사이트맵 또는 주요 화면 관계도.

### 3.3. 인터랙션 원칙 (Interaction Principles)
- **피드백:** 사용자 행동에 대한 즉각적이고 명확한 시스템 피드백 제공 (예: 버튼 클릭, 데이터 저장 성공/실패).
- **오류 방지 및 처리:** 사용자의 실수를 최소화하는 설계, 오류 발생 시 명확한 원인과 해결 방법 안내.
- **일관된 제어:** 유사한 기능에는 일관된 인터랙션 방식 적용.
- **사용자 주도:** 사용자가 시스템을 통제하고 있다고 느낄 수 있도록 설계.

### 3.4. 접근성 (Accessibility - A11Y)
- **준수 목표:** 한국형 웹 콘텐츠 접근성 지침(KWCAG) 2.1 AA 수준 준수 목표.
- **주요 지침:** 키보드 접근성, 스크린 리더 호환성, 명도 대비, 이미지 대체 텍스트, 명확한 링크 텍스트 등.

## 5. UI 컴포넌트 라이브러리 (Component Library / Pattern Library)
(각 컴포넌트별 상세 명세: 시각적 디자인, 크기/여백 등 스펙, 상태별 변화, 인터랙션, 사용 가이드라인, Do & Don't 예시 포함)

### 5.1. 버튼 (Buttons)
- 종류: Primary, Secondary, Tertiary/Text, Outline, Destructive, Icon Button 등.
- 크기: Large, Medium, Small.
- 상태: Default, Hover, Focused, Active/Pressed, Disabled.

### 5.2. 입력 필드 (Input Fields)
- 종류: Text, Password, Number, Date Picker, Text Area, Search Input.
- 상태: Default, Focused, Error, Disabled, Read-only.
- 구성요소: Label, Placeholder, Helper Text, Error Message, (선택) Prefix/Suffix Icon.

### 5.3. 선택 컨트롤 (Selection Controls)
- 드롭다운 (Dropdowns / Selects): 기본, 검색 가능.
- 체크박스 (Checkboxes): 단일, 그룹.
- 라디오 버튼 (Radio Buttons).
- 스위치/토글 (Switches/Toggles).

### 5.4. 네비게이션 (Navigation)
- GNB (Global Navigation Bar) / 상단 메뉴.
- LNB (Local Navigation Bar) / 사이드 메뉴.
- 탭 (Tabs): 기본 탭, 세로 탭, 박스형 탭 등.
- 브레드크럼 (Breadcrumbs).
- 페이지네이션 (Pagination).

### 5.5. 데이터 표시 (Data Display)
- 테이블 (Tables): 기본 스타일, 정렬, 필터링 연동, 행 선택.
- 목록 (Lists): 기본 목록, 설명 목록.
- 카드 (Cards): 정보 요약 및 그룹화.
- 배지/태그 (Badges/Tags): 상태, 카테고리, 알림 수 표시.
- 툴팁 (Tooltips): 부가 정보 제공.

### 5.6. 피드백 및 알림 (Feedback & Notifications)
- 모달/팝업 (Modals/Popups): 확인, 경고, 오류, 정보 제공, 입력 폼.
- 토스트/스낵바 (Toasts/Snackbars): 간단한 상태 변경 알림.
- 인라인 알림/경고 메시지 (Inline Alerts/Warnings).
- 로딩 인디케이터 (Loading Indicators): 스피너, 프로그레스 바, 스켈레톤 UI.
